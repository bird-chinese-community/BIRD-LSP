# 实施规划师

创建全面的项目实施计划，包含详细的代码结构、类型系统、构建流程和测试方法。

## 使用场景

- 规划开发阶段和里程碑
- 设计代码组织结构
- 创建类型定义和接口
- 设置构建和部署流程
- 建立测试策略
- 制定编码规范

## 规划流程

### 1. 代码结构设计

定义项目组织：

```
src/
├── core/              # 核心业务逻辑
├── adapters/          # 外部适配器
├── ports/             # 接口定义
├── infrastructure/    # 基础设施
└── utils/             # 工具函数
```

### 2. 并行规划（Sub-agents）

启动 4+ 个子代理：

**代理 1：代码结构**
```
定义：
- 目录结构
- 文件职责
- 导入/导出关系
- 模块边界
- 编码规范
```

**代理 2：类型系统**
```
设计类型：
- 领域模型类型
- API 契约类型
- 配置类型
- 错误类型
- 类型守卫
```

**代理 3：构建与部署**
```
设计构建流程：
- 构建工具配置
- 编译选项
- 打包策略
- 部署流程
- CI/CD 集成
```

**代理 4：测试策略**
```
规划测试：
- 单元测试
- 集成测试
- E2E 测试
- 测试数据
- 覆盖率目标
```

### 3. 阶段规划

分解为开发阶段：
- **阶段 1**：基础搭建
- **阶段 2**：核心实现
- **阶段 3**：功能完善
- **阶段 4**：测试优化

## 交付物

1. **项目结构文档**：
   - 完整目录树
   - 模块规范
   - 依赖关系

2. **类型定义文档**：
   - 类型接口
   - 验证模式
   - 类型守卫

3. **构建工作流文档**：
   - 构建配置
   - 开发脚本
   - 部署流程

4. **测试策略文档**：
   - 测试金字塔
   - 测试示例
   - CI/CD 集成
