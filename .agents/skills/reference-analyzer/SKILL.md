# 参考项目分析器

分析现有软件项目，提取架构模式、技术选型和最佳实践，用于新项目开发或技术决策。

## 使用场景

- 技术选型调研（框架、库、工具链）
- 比较多个参考实现
- 评估架构模式（单体 vs 微服务、同步 vs 异步等）
- 识别现有方案之间的功能差距
- 提取配置模式和约定
- 规划构建系统和 CI/CD 流程

## 分析维度

### 1. 架构与结构
- 项目组织方式
- 模块边界和依赖关系
- 分层架构设计
- 数据流和通信模式

### 2. 技术栈
- 核心框架和库
- 构建工具和打包器
- 测试框架
- 部署和运维工具

### 3. 代码质量
- 编码规范
- 类型系统使用
- 错误处理模式
- 日志和监控

### 4. 工程实践
- 版本控制策略
- CI/CD 流程
- 文档规范
- 代码审查流程

## 分析工作流

### 1. 确定参考项目

选择 2-4 个代表性项目：
- **同类项目**：相似领域或功能
- **成熟项目**：经过生产验证
- **不同方案**：展示多种解决思路

### 2. 并行分析（Sub-agents）

启动 4+ 个职责不同的子代理：

**代理 1：架构分析**
```
分析项目结构：
- 目录组织
- 模块边界
- 依赖关系图
- 架构模式（MVC、微服务、事件驱动等）
```

**代理 2：技术栈分析**
```
分析技术选型：
- 核心依赖和版本
- 构建工具链
- 测试框架
- 部署方案
```

**代理 3：代码质量分析**
```
分析代码实践：
- 编码规范
- 类型安全
- 错误处理
- 性能优化
```

**代理 4：工程实践分析**
```
分析开发流程：
- Git 工作流
- CI/CD 配置
- 文档结构
- 代码审查
```

### 3. 综合对比

整合子代理输出：
- 对比矩阵
- 架构推荐
- 技术选型决策树
- 最佳实践清单

## 输出格式

生成包含以下内容的分析报告：
1. 执行摘要（关键洞察）
2. 详细对比表
3. 架构模式推荐
4. 技术选型建议
5. 功能差距分析
