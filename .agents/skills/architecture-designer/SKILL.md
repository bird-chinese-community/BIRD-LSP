# 架构设计师

设计全面的软件系统架构，关注模块化、可维护性和扩展性。

## 使用场景

- 新系统的架构设计
- 现有系统的架构重构
- 定义模块边界和接口
- 设计配置管理系统
- 规划技术演进路线
- 制定扩展策略

## 设计流程

### 1. 需求分析

识别核心需求：
- 功能需求
- 非功能需求（性能、安全、可扩展性）
- 约束条件
- 集成点

### 2. 架构决策

记录关键决策：
- 架构风格（分层、微服务、事件驱动等）
- 技术选型
- 数据存储策略
- 通信机制

### 3. 模块化设计（Sub-agents）

启动 4+ 个子代理并行设计：

**代理 1：系统结构**
```
设计系统组织：
- 分层架构
- 模块边界
- 职责划分
- 依赖方向
```

**代理 2：组件设计**
```
设计核心组件：
- 组件接口
- 生命周期管理
- 错误处理策略
- 状态管理
```

**代理 3：配置与扩展**
```
设计配置系统：
- 配置分层
- 环境管理
- 扩展点设计
- 插件机制
```

**代理 4：集成与边界**
```
设计集成策略：
- 外部系统接口
- 数据交换格式
- 认证授权
- 边界上下文
```

### 4. 定义契约

明确接口和契约：
- API 定义
- 数据模型
- 事件格式
- 错误规范

## 交付物

1. **架构设计文档**：
   - 系统结构图
   - 模块依赖图
   - 组件交互图
   - 数据流图

2. **接口规范**：
   - API 定义
   - 数据模型
   - 事件契约

3. **决策日志**：
   - 架构决策记录（ADR）
   - 权衡分析
   - 风险评估
